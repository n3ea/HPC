# LR0: Matrix Multiplication CPU vs GPU 



## Технические характеристики

### Аппаратное обеспечение
- **CPU**: Intel(R) Core(TM) i7-14700KF (3.40 GHz)
- **GPU**: NVIDIA GeForce RTX 5070
- **CUDA version**: 13.2.0


## Результаты тестирования

### Время выполнения и ускорение

| Размер матрицы | CPU (сек) | GPU (сек) | **Ускорение (x)** |
|----------------|-----------|----------|-------------------|
| 100x100 | 0.1540 | 0.000370 | **416.22** |
| 200x200 | 1.1934 | 0.000375 | **3182.40** |
| 500x500 | 18.9222 | 0.003490 | **5422.35** |
| 800x800 | 78.7539 | 0.006206 | **12689.56** |
| 1000x1000 | 160.2240 | 0.013873 | **11549.83** |
| 2000x2000 | 1247.5602 | 0.048922 | **25499.18** |


## Анализ результатов

### 1. Производительность на малых размерах (100x100 - 200x200)

**Наблюдение:**
- При размере 100×100 ускорение составляет 416x
- При размере 200×200 ускорение возрастает до 3182x

**Причина:**
На малых размерах матриц накладные расходы на передачу данных на GPU и запуск ядра составляют значительную часть общего времени. Однако даже с этими накладными расходами GPU уже показывает впечатляющее преимущество благодаря массовому параллелизму.

### 2. Производительность на средних размерах (500x800 - 1000x1000)

**Наблюдение:**
- 500×500: ускорение 5422x
- 800×800: ускорение 12690x (пиковое значение)
- 1000×1000: ускорение 11550x (небольшое снижение)

**Причина:**
При размере 800×800 достигается оптимальная загрузка GPU. Количество вычислительных операций (N³) становится достаточно большим, чтобы полностью загрузить все ядра GPU. При 1000×1000 наблюдается небольшое снижение ускорения, что может быть связано с ограничениями кэш-памяти или началом влияния накладных расходов на управление памятью.

### 3. Производительность на больших размерах (2000x2000)

**Наблюдение:**
- Ускорение достигает рекордного значения **25499x**
- CPU время: 1247.56 сек 
- GPU время: 0.049 сек 

**Причина:**
При размере 2000×2000 количество операций достигает 8×10⁹. На таком объеме вычислений преимущества параллельной архитектуры GPU раскрываются максимально. CPU выполняет операции последовательно, в то время как GPU обрабатывает тысячи элементов одновременно, что приводит к колоссальному выигрышу в производительности.




## Выводы

1. **GPU демонстрирует колоссальное преимущество** в задачах умножения матриц, особенно на больших размерах. Ускорение достигает 25499 при размере 2000×2000, что сокращает время вычислений с 20.8 минут до 49 миллисекунд.

2. **Параллельная архитектура GPU** позволяет эффективно решать задачи, которые на CPU требуют значительных временных затрат. Это делает GPU незаменимым инструментом для научных вычислений, машинного обучения и обработки больших данных.

3. **Эффективность использования GPU растет с увеличением размера задачи**. На малых размерах (100×100) ускорение составляет 416, на средних (800×800) - 12690, на больших (2000×2000) - 25499.

4. **Время выполнения на GPU остается практически константным** при росте размера матрицы от 100 до 200 (0.37 мс), в то время как CPU время увеличивается с 0.154 до 1.193 секунд.
