## Лабораторная работа №2: Медианный фильтр 3x3 с использованием CUDA (сравнение CPU/GPU)

Реализовать 3x3 медианный фильтр для удаления шума "соль и перец" с изображений. Алгоритм выполняется как на CPU (последовательная версия), так и на GPU (параллельная версия на CUDA) с последующим сравнением производительности.

### Результаты ускорения

На тестовом изображении размером **512 x 512** пикселей были получены следующие результаты:

- **Время выполнения на CPU:** 1.293154 сек
- **Время выполнения на GPU:** 0.000091 сек
- **Ускорение:** 14163.79


### Почему медианный фильтр?

Медианный фильтр относится к классу нелинейных фильтров и обладает важными преимуществами:

- **Сохранение границ:** В отличие от усредняющего (линейного) фильтра, медианный фильтр не размывает резкие перепады яркости — края объектов остаются четкими.
- **Эффективность против импульсного шума:** Шум "соль и перец" (случайные белые и черные пиксели) практически полностью удаляется, так как выбросы не попадают в медиану окрестности.
- **Простота реализации:** Алгоритм не требует сложных математических вычислений — только сортировка и выбор среднего элемента.

### Как работает алгоритм

Для каждого пикселя входного изображения выполняются следующие шаги:

1. **Извлечение окрестности 3×3:** Собираются значения 9 соседних пикселей (включая сам центральный пиксель).
2. **Обработка границ:** Если окно выходит за пределы изображения, недостающие пиксели заменяются ближайшими существующими (clamping).
3. **Сортировка:** 9 значений упорядочиваются по возрастанию.
4. **Выбор медианы:** Пятый элемент отсортированного массива (центральный) записывается в результирующее изображение.

### Параллельная модель
- Один поток = один пиксель: Каждый поток GPU независимо обрабатывает свой пиксель, что дает идеальное распараллеливание.

- Размер блока: 16×16 = 256 потоков на блок. Это стандартный выбор, обеспечивающий хорошую загрузку ядер GPU.

- Размер сетки: Для изображения 512×512 сетка составляет 32×32 блока, всего запускается 1024 блока × 256 потоков = 262 144 потока.

### Зависимости

Для работы программы необходимы следующие библиотеки Python:

- `numpy` — для работы с массивами изображений
- `Pillow` (PIL) — для загрузки и сохранения изображений
- `pycuda` — для взаимодействия с CUDA на Python
- `matplotlib` — для визуализации результатов
