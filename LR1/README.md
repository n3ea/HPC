## Реализация

### 1. Алгоритм на CPU (последовательный)

```python
def vector_sum_cpu(vector):
    total = 0
    for elem in vector:
        total += elem
    return total
```

#### Что было распараллелено

- **Распределение элементов вектора между потоками GPU** — каждый поток обрабатывает один элемент
- **Параллельная редукция внутри каждого блока через shared memory** — потоки внутри блока попарно суммируют свои значения
- **Финальное суммирование результатов блоков с использованием атомарных операций** — результаты от всех блоков складываются в глобальную сумму

#### Почему именно так
- **Параллельная редукция** позволяет выполнить суммирование внутри блока за O(log n) шагов вместо O(n), что значительно ускоряет вычисления
- **Shared memory** обеспечивает быстрый обмен данными между потоками внутри блока (доступ к shared memory значительно быстрее, чем к глобальной памяти)
- **Атомарные операции** безопасно суммируют результаты от разных блоков, исключая состояние гонки при одновременном доступе к одной ячейке памяти
- **Размер блока 256 потоков** — оптимальное значение для большинства современных GPU, обеспечивающее эффективную загрузку вычислительных ядер
